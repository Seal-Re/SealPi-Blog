# 管理写接口与鉴权（v1）

本文档用于快速说明后端一期（v1）已落地的“管理写接口（草稿/发布）”与“/api/v1/admin/** 鉴权拦截”的使用方式、配置项与本地验证步骤。

## 1. 已实现内容

- 管理写接口（JSON）：保存草稿 / 发布
  - 创建：POST `/api/v1/admin/articles?action=draft|publish&coverImageUrl=...`
  - 更新：PUT `/api/v1/admin/articles/{id}?action=draft|publish&coverImageUrl=...`
- 鉴权拦截：保护 `/api/v1/admin/**`
  - Header：`Authorization: Bearer <token>`
  - Token：HS256 JWT（后端离线验签）
  - 白名单：GitHub userId 命中 `admin.github.userIds`

## 2. 接口说明

### 2.1 创建（保存草稿 / 发布）

- Endpoint：POST `/api/v1/admin/articles`
- Query：
  - `action`：`draft`（默认）或 `publish`
  - `coverImageUrl`：可选；预览图 URL（单封面，用于 OG/list 兜底）
- Body（JSON）：见 DTO [`ArticleDraftSaveCmd`](v1/../sealpi-blog/blog-client/src/main/java/com/seal/blog/client/article/dto/cmd/ArticleDraftSaveCmd.java:13)

### 2.2 更新（保存草稿 / 重新发布）

- Endpoint：PUT `/api/v1/admin/articles/{id}`
- Query：同创建
- Body（JSON）：见 DTO [`ArticleDraftUpdateCmd`](v1/../sealpi-blog/blog-client/src/main/java/com/seal/blog/client/article/dto/cmd/ArticleDraftUpdateCmd.java:11)
- 说明：路径参数 `{id}` 会覆盖 DTO 中的 `articleId`，以路径为准。

## 3. 鉴权与配置

### 3.1 鉴权逻辑

- Filter：[`AdminAuthFilter`](../sealpi-blog/blog-adapter/src/main/java/com/seal/blog/adapter/security/AdminAuthFilter.java:18)
  - 仅拦截 `/api/v1/admin/**`
  - 校验 `Authorization` 是否为 `Bearer <token>`
  - 调用校验器验签 + 白名单检查

- 校验器：[`AdminJwtVerifier.verifyAuthorizationHeader()`](../sealpi-blog/blog-adapter/src/main/java/com/seal/blog/adapter/security/AdminJwtVerifier.java:34)
  - 仅校验签名（HS256）
  - 从 payload JSON 中提取 claim（默认 `githubUserId`）
  - 与 `admin.github.userIds`（CSV）比对

### 3.2 配置项

配置读取位置：[`AdminAuthConfig`](../sealpi-blog/blog-adapter/src/main/java/com/seal/blog/adapter/config/AdminAuthConfig.java:11)

- `admin.jwt.secret`：JWT HS256 对称密钥（必填）
- `admin.github.userIds`：管理员 GitHub userId 白名单，CSV（必填）
- `admin.jwt.githubUserIdClaim`：JWT payload 中 userId 的 claim 名（默认 `githubUserId`）

示例（建议放到 `blog-adapter` 的 `application.properties` 或启动模块的配置中）：

```properties
admin.jwt.secret=REPLACE_ME
admin.github.userIds=12345,67890
admin.jwt.githubUserIdClaim=githubUserId
```

## 4. 本地验证

1) 运行测试：

```bat
cd sealpi-blog
mvnw.cmd -q test
```

2) 运行应用（以实际启动模块为准），然后使用 curl 或 Postman 调用：

- 无/错 token 访问管理接口应返回 401
- token 签名正确但 userId 不在白名单应返回 403
- token 正确且白名单命中应放行到 Controller
